\chapter{緒言}

	\section{研究の背景}
	\,本研究室は毎年高専ロボコンに参加している.これまでに三年連続,全国大会に出場しており,
	今年は地区大会で優勝すること目指した.
	
	本大会は例年より,アイデア対決がテーマにあるので,面白要素やデザイン性,オリジナルな機構など,
	ロボットの重点は毎年どのチームも異なる.今回の地区大会では優勝することを目標にした私たちは,
	それに向けてロボットを作成してきたが今回は制作などに時間がかかっており,
	本来の完成予定日を大幅に過ぎ,練習の時間が削れてしまい,
	初戦の競技を行っている最中に一台が操縦ができなくなるという今までに起きなかった問題が起きて負けてしまった.
	
	この結果より考察すると,なにが完成予定日のズレの原因か調べる必要がある.
	
	\section{研究の目的}
	\,我々が普段から日報を書く意味として,全員毎日書いている日報の
	「問題点・新たに知ったこと・できたこと」から,
	その日の作業進捗状況が分かる.そのため今回はその日報から,
	学生が知りたい問題を抽出し,各作業にどのくらいの時間がかかっているのかを
	見える形にすることで作業効率の低下の要因を分析できるようにする.

	１章では，研究目的や高専ロボコンを考察したものである．

\chapter{原因分析}

	\section{ズレの原因分析}
	原因を分析するにあたって日報のテキストマイニングを用いる.
	7月からすでに遅れが出ているため7月の行動が全体の遅れに関係していると思われる.
	\begin{table}[H]
	\caption{7月の日報}
	\begin{center}
	\centering
	\includegraphics[width=80mm]{キャプチャ５.PNG}
	\end{center}
	\label{table:j}
	\end{table}
	上記の表\ref{table:j}を見て,7月から遅れだしたと仮定するならば,
	テキストマイニングの結果,「加工」,「部品」,「モータ」
	の出現頻度が高くスコアも高いので.今回はこの3つに絞って分析する,
	ちなみに今回「モータ」は「モータドライバ」として日報に書かれているため,
	「モータドライバ」として記述する.
	
	
	まず「部品」を日報から分析すると以下のようなことが分かった.
	\begin{itemize}
		\item 部品がまだ届いていなかった.
		\item PCでCADを開かないと製作に必要な部品がわからなかった.
		\item 設計でまだ完全に終わっていない部品が少しある.
		\end{itemize}
		
	次に「加工」を日報から分析すると以下のことが分かった.
	\begin{itemize}
		\item フライス加工での切込量を少なくすると時間がかかった.
		\item 加工手順が分からず部品が製作できなかった.
		\item 加工をする際に工具の場所がわからなかった.
		\end{itemize}
	
	次に「モータドライバ」を日報から分析すると以下のことが分かった.
	\begin{itemize}
		\item 腕のモータドライバの不具合がある.
		\item 駆動部のモータドライバが燃える.
		\item 腕のモータドライバのコンデンサが破裂.
		\end{itemize}
	
	スケジュールが後ろに押された原因として以上のことが関係していると思われる.
	
	
	なので日々の授業や課題での経験や知識が必要だと考えられる.

	\section{考察}
	\,以上で述べた事から,予定の大幅なズレの原因としては以下のものが考えられる.
	
	\begin{itemize}
		\item 注文する部品の選定.
		\item 部品の加工に対する情報収集や知識不足.
		\item 電気系の知識.勉強不足による試行錯誤の連続.
		\item プログラムの不具合の修正.
		\end{itemize}
	
	また,実際に体験したもので
	\begin{itemize}
		\item 夢工房までの移動時間.
		\item 機械の工具の取り付け.
		\item ライセンス講習による機械の使用禁止.
		\item 各班員の所用,風邪などによる欠席.
		\end{itemize}
	スケジュール的には上記のものを改善できれば十分な練習時間を確保することが出来ると思われる.
	今回のロボット製作のスケジュールを立てた際.
	これらの多くが考慮されていなかったため今回のスケジュールは不適切だったと考えられる,
	実際に自分たちが経験して,注文の仕方,電気系の経験不足を感じた.
	また.今回全員が書いていた日報は後からデータ化して整理するときに書いてあることがすぐに区別しやすいため,
	大会で優勝するのに必要なツールと言える,
	基本的に何時まで作業して,残業は何時までと決めて,調整と練習を同時に行うと,幾何か時間をとれると
	思われる.

		\section{電気系}
		\begin{enumerate}
		\item 信号がノイズに弱いから,信号線と電源線を分けて配線する.
		
		\item 部品の名前をいち早く覚える.全員がある程度把握していると人に発注を頼んだ時や問題の目星を付けやすい.
		
		\item 電線の太さに気をつける.大電流が流れる線には太いものを用いる.
		
		\item 信号線はねじってアルミホイルで包んでおくことで,ノイズが発生しにくくなる.
		
		\item モータの付近等,機械部品の近くで線を切るときは,線の切れ端が機械に巻き込まれないように注意する.モータの中に入るとモータが壊れる.
		
		\item ロボットの主電源を入れる前に（バッテリーをつなぐ前に）必ず,テスターで短絡確認を行う.
		
		\item ヒューズ管を用いると,全体の回路が壊れるのを阻止できる可能性ができる.図\ref{fig:12}に使ったガラスヒューズ管を示す.
		
		\item はんだ吸い取り器を用いるときは,ランドごと吸い取らないように気を付ける.
		
		\item ランドの温度をあげないと,はんだが早く溶けない.図\ref{fig:13}にハンダ付けの様子を示す.

		\subsection{データ系}
		\begin{enumerate}
		\item 図面には加工に必要な情報を入れる.
		
		\item 寸法に変更があった場合加工者が加工に行く前に,図面に記入しておく.
		
		\item 日報はより細かくその日の事柄を書かないと,後でまとめる時まとめられない.
		
		\item 空白はデータ量が減るため,何かしら書いた方がよい.
		
		\item 日報に関係のないことがかかれているものがある.
